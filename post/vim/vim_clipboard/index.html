vim 中的两个剪贴板寄存器

之前用的一些 Terminal emulator，比如 deepin terminal gtk. gnome terminal,本身都做了一些特性 来让 vim 和系统的剪贴板直接交互，比如在 vim 中直接选中 ctrl+shift+c/v，或者右键选中复制粘贴 (当然这些需要 disable mouse)，所以我之前从来没有遇到如何从 vim 中复制/粘贴内容到其他软件内的问题， 也就从来不知道 vim 是有两个剪贴板的，不知道 vim 的 * + 寄存器的作用

这次从 xorg 切换到 wayland，发现 deepin terminal 无法在 wayland 下运行，就先用 gnome terminal 将就了几天，等闲下来了(指 Steam 秋促买的游戏玩腻了)，先从目前比较火的三个支持 gpu 加速的终端模拟器中一个一个配置并试用了几天之后，确认了 wezterm，作为我的下一个终端模拟器

先说说这三个终端模拟器的个人感受

  1. alacritty

    确实简洁，而且确实很快，配置文档也挺简单，但是我翻来覆去没有找到如何在一个 window 里面拆分窗格， 后来在项目的 Issue 里面找到了答案， 作者意不在此，而且目前看起来也不打算做 ，可惜我对拆分窗格的需求还挺高的，所以，没办法，放弃

  2. kitty

    这个其实我并没有看完文档，在我安装之后，看着文档配置了10分钟我就放弃了，我能感受到它很强大， 自定义性很高，但是对我来说，自定义性有些太高了，一个很简单的功能，可以有一整页的配置选项， 在如此繁多的文档内容中找到我需要的，让我头都大了，遂放弃（这个是支持拆分窗格的，并且有很多布局，很多可配置选项

  3. wezterm

    其实我一开始对 wezterm 是没有抱太大期望的，因为之前无意中看到它的一个 Issue，有人抱怨 wezterm 和前面两个比起来渲染速度慢太多了，虽然下面有作者对此的解释，说那些测试方法不太标准，wezterm 是交互延迟最低的等等，但当时给我留下了一个不太好的印象，不过，在我最后安装并看了文档开始配置好使用的时候， 才发现 wezterm 是最适合我的，文档很丰富，简洁明了，自定义个人认为虽然比不上 kitty 可配置项那么多， 但是我需要的都有，并且给了一堆的 api，你如果有其他需求可以自己去实现，它可能在某些方面不是最强的，但是是最适合我的

说回 vim，其实换一个终端模拟器对 vim 应该是没什么影响的，wezterm 也有 key binding 和 mouse binding， 来实现其他功能，当然包括了从 vim 中复制粘贴内容到外部和其他软件交互，直到今天，我想从某个文件中复制好几页的内容出来， 但是突然发现 wezterm 在 vim 里无法滚屏选择来复制，也就是最多也只能复制一整页的内容，如果 enable mouse 的话， 当然可以滚屏选择，但是这个时候在 vim visual mode 下，wezterm 的 key binding 和 mouse binding 的功能就失效了，无法复制，所以，在使用了这么久的 vim 之后，我第一次搜索了如何从 vim 中复制内容到外部……

搜索到的文章大多也都说了，需要编译时启用 +clipboard，我使用的是 neovim， 看了下官方文档，neovim把这些功能实现都托付给了其他应用程序来实现， 称之为 provider，clipboard 这部分当然也是，需要先安装一个clipboard tool，根据官方文档的介绍， 我安装了 xclip，然后查看 checkhealth 的 clipboard 部分，已经识别到了 xclip，echo has('clipboard') 也返回了 1 ，但是这个时候在 vim 中复制剪贴的内容还是无法在外部使用，因为 vim 默认复制删除剪贴等操作默认使用的是 vim 的 " 寄存器，也叫做 unnamed 寄存器，是 vim 的默认寄存器，而与系统剪贴板对应的是 + 寄存器， 在 Windows 和 MacOS 中，他们只有一个 clipboard，+ * 寄存器是一样的，但在基于 x11 的系统上， 比如 Linux 和大部分 Unix like 系统中，是有两个 clipboard 的，分别是

  1. Primary : * 寄存器对应的剪贴板，可以使用鼠标选中内容，使用鼠标中键粘贴

  2. Clipboard : + 寄存器对应的剪贴板，Linux 下通常使用 Ctrl+Shift+c/v/x 来复制粘贴剪切等

如果想让 vim 使用系统剪贴板，则需要配置一下，让 vim 使用+寄存器即可，这样，之后在 vim 中复制粘贴 的内容都会与系统剪贴板同步了，就可以从 vim 中复制内容到外部了

配置方法：

在.vimrc/init.vim 中加入

1

set clipboard=unnamedplus
