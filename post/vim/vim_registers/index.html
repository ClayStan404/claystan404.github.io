vim 中的寄存器

因为前面文章中说的研究 vim 剪贴板导致，接触到了 vim 寄存器这个概念,发现自己对这方面也是一无所知，所有有了这篇文章

其实很多人在使用了很久 vim 之后都对 vim 的寄存器没有什么概念，当然这点也是我在搜索资料是看到下面的人评论了解到的 我本来还听不好意思的，表面说着自己是 vim 用户，但是对 vim 什么都不了解，但是看到下面评论有个老哥说自己用了 10 年 vim 也不知道，突然就有点没那么不好意思了嘿嘿(毕竟它是编辑器之神

什么是 vim 寄存器

vim 寄存器是内存中的空间，vim 用来存储一些文件和操作，每个寄存器都有一个标识符，以便访问

如何访问寄存器

一般情况下，除了一些特殊的寄存器

在 normal mode 下，使用 "寄存器标识符 来访问一个寄存器 比如 "a 表示访问 标识符为 a 的寄存器，"ayy 表示将当前行的内容复制到 a 寄存器中， "ap 表示将标识符 a 中的内容粘贴到鼠标光标后

在 insert mode 下，按下 Ctrl+r，然后按寄存器标识符 来访问一个寄存器，比如按下 Ctrl+r 后，再按 a ，访问 a 寄存器

寄存器的种类

  1. 未命名寄存器

    " : 可以称之为 vim 的默认寄存器

    因为你所做的任何操作如果没有显式指定寄存器，则默认使用这个寄存器， 标识符为 "，所以可以通过 "" 来访问这个寄存器 你做的任何复制，删除(d c s x y)等操作的文本都会放在这里，可以通过 ““p 来复制 该寄存器中的内容

  2. 0-9的数字寄存器

    0 : 寄存器将保存最近一次复制(yank)的内容

    1-9 : 依次保存的最近9次删除的内容(要求删除的文本不少于一行)

  3. 小删除寄存器

    - : 保存最近依次删除的内容不超过一行的内容，区别与上面 1-9 寄存器

  4. 26个字母命名的寄存器

    a-z: 共 26 个寄存器

    跟前面的不一样，这些官方未预定义内容，交由用户自定义使用 大小写表示同一个寄存器，但是大小写在使用时行为不同，如果将一些内容写入小写的名称 中时，会替换原有的内容，如果使用大写的名称，则是向原来的寄存器中附加内容

    可以和 vim 的 recording 功能合作使用，保存一些操作来重复使用

  5. 只读寄存器

    % : 内容为当前编辑的文件的路径

    . : 内容为你最近一次插入的文本

    : : 最近依次执行的命令

    这些寄存器的内容无法被人为显式写入，仅能读取

  6. 备用文件寄存器

    # : 如果打开多个文件，内容为另一个文件的路径

  7. 表达式寄存器

    = : 可以在 vim 中计算表达式的结果

    在 normal mode 中，可以通过 “=AnExpression 来保存该表达式的结果 然后可以通过 :put = 来打印结果

    在 insert mode 中，按下 Ctrl+r=AnExpression 来计算结果然后自动打印出来

  8. 搜索寄存器

    / : 保存最新一次通过 / 搜索的文本内容

  9. 选择-丢弃寄存器

    * + : 选择寄存器

    这个在前面的文章中介绍过，*对应 Primary ，+对应 Clipboard

    ~ : 删除寄存器，仅适用于 gvim，保存上次拖放操作的文本

  10. 黑洞寄存器

_ : 如果想更改/删除文本而又不希望当前其他寄存器的内容更改时，显式的使用该寄存器即可

查看寄存器中的内容

在 vim 的 cmd line mode 中使用 reg 命令即可查看所有寄存器的内容

如果仅查看个别寄存器的内容，指定标识符即可: reg " 7
